# AI 口语陪练 UX 重构设计 (P0)

- 日期:2026-06-06
- 状态:设计定稿,待实施计划
- 关联 mockup:`docs/mockups/voice-coach-ux-mockup.html`、`docs/mockups/voice-coach-ux-mockup-v2.html`

## Context(为什么做)

当前应用已有场景卡片、学习路径、虚拟人物和练习页面,但整体偏"功能堆叠":用户进首页不知道第一步做什么;虚拟人物像装饰;练习页保留强烈的"文本输入/提交"痕迹,不像真实口语对话;评分只展示结果,缺少过程正反馈;页面之间没有"课前目标→课中陪练→课后报告→下一轮"的学习闭环;也缺少留存与商业化结构。

本次把界面从"功能展示页"升级为**"低压开口 · 真实追问 · 可见成长"**的学习陪练产品体验。视觉语言不重做,**复用现有品牌系统**(`src/index.css` 的 `--feather-green` 等 token 与 `src/components/BrandGuidelines.tsx`)。

## 三条设计原则

1. **教育学**:每页围绕单一、可衡量的学习目标。课前告诉用户本轮只练一件事;课中围绕该目标持续追问、不频繁打断纠错;课后给"完成了什么 + 最该改的一句 + 下一轮只练什么"。
2. **心理学**:降低开口焦虑。去考试化文案(用"练习/提示/可以更自然/本轮重点/成长建议",不出现"测试/错误/扣分/失败/能力不足");即时正反馈结构固定为 **先肯定 → 指出一个改进点 → 给一句可直接复用表达**;收起技术链路信息降低认知负担。
3. **商业产品**:突出"真实场景追问 / 低压开口 / 课后报告 / 持续成长"价值;成长轨迹形成留存;功能分层预留付费扩展位(UI 不强推付费)。

## 决策记录

| 决策点 | 选择 |
| --- | --- |
| 视觉语言 | 不重做,复用现有品牌 token 与组件 |
| 虚拟人物风格 | **B 圆润吉祥物**(大头大眼、亲和力强,贴合低压定位) |
| 人物落地方式 | 先用**升级版 SVG + CSS 动画**占位,预留 Lottie 替换接口(不需额外素材) |
| 场景 | 保留 3 种(面试/点餐/会议)+ 支持「＋自定义」 |
| 练习页输入 | 舞台式语音交互,弱化大文本框(保留小字幕) |

## 页面设计

### ① 首页 — 三栏 hero + 价值卡 + 场景卡

- **左「今日任务」**:低压主标题(例:"今天只练 5 分钟,把一个回答说清楚");副标题("AI 会先提问,你只需要开口回答。说错也没关系,系统会在课后帮你整理");目标盒(本轮单一目标);主按钮「开始 5 分钟练习」;辅助按钮「更换场景 / 查看上次报告」。
- **中「虚拟陪练」**:吉祥物人物放大居中(260–320px)+ 欢迎气泡("Ready for a 5-minute interview practice?")+ "待机陪练中"状态。
- **右「成长轨迹」**:连续练习天数、累计口语时间、最近得分、当前薄弱项、推荐下一练 + 下一步建议。
- **价值卡(4)**:真实场景追问 / 低压开口 / 课后成长报告 / 持续成长。
- **场景卡**:每张含训练目标、核心能力、适合人群、时长、难度、标签(推荐/适合面试前/低压力入门等);加「＋自定义」卡。

首屏信息优先级只保留三件事:今天练什么、为什么练、点哪里开始。

### ② 练习页 — 舞台式语音交互

- **顶部**:场景名 · 当前轮次 · 倒计时 · 本轮目标。
- **中央舞台**:**按场景填充背景**(面试间/餐厅/会议室的插画背景)+ 聚光;吉祥物人物居中;AI 问题气泡;圆形主按钮「开始回答 / 正在聆听 / AI 正在思考」;聆听时脉冲环;底部实时字幕。检测到静音后自动提交(P0 先做按钮显式提交 + 字幕,自动 VAD 归 P1)。
- **右侧导航**:当前目标 + 完成条件;本轮轻提示(围绕目标,如"试着加入一个数字");实时成长指标;本轮状态清单(场景目标/实时对话/发音评测/表达纠错/课后总结);技术链路默认折叠为"🟢 语音链路正常",展开才显示 ASR/LLM/TTS 耗时。
- **取消左侧重复小人**,人物统一放中央舞台。
- **人物状态**:idle → listening → thinking → speaking。

### ③ 报告页 — 学习诊断单

顺序:总分 + 本次完成目标 + 一句鼓励 → 五维能力评分(发音/流利度/语法/词汇/互动完成度)→ "最值得改的一句"(Original → Better → 原因)→ 下一轮单一目标 → 推荐复练句 → 正向收尾文案。即使分数一般也强调"下一步可执行",不打击用户。

## 学习闭环

课前(设单一目标)→ 课中(围绕目标追问 + 即时正反馈)→ 课后(诊断单 + 下一轮目标)→ 回首页(成长轨迹更新 + 推荐下一练)。

## 技术实现要点(基于现有代码)

- **人物组件**:升级 `src/components/CoachAvatar.tsx` 为吉祥物 SVG,接收 `state: idle|listening|thinking|speaking`,内部用 CSS 动画;封装为单一接口,后续可替换为 Lottie 而不动调用方。
- **语音播放**:复用已修好的 `playSpeech`(`src/App.tsx`)真实音频优先、合成音兜底的逻辑。
- **自定义场景**:复用 `src/storage.ts` 的 `saveCustomScenario / loadCustomScenarios` 与现有自定义表单。
- **本轮目标 / 即时反馈**:后端 `server/providers/liveProviders.ts` 的 dialogue 返回结构里增加 `roundGoal`、`positiveFeedback` 字段(schema 在 `shared/schemas.ts`),mock provider 同步;前端围绕目标渲染右侧导航与反馈。
- **成长轨迹**:P0 用 **mock/示例数据**渲染(连续天数、累计时长、最近分、薄弱项、推荐下一练),保留 UI 与设计,**不接真实持久化**;真实数据来源(`src/domain/learning.ts` / 后端)留待后续。
- **去压力化文案**:集中到一处文案常量(便于审校),报告/练习页统一引用。
- **页面结构**:首页/练习页按三栏 + 舞台重排,沿用现有 CSS 变量与卡片样式。

## 验收标准

1. 进首页 3 秒内知道今天练什么。
2. 不理解技术链路也能开始练习(链路默认折叠)。
3. 主视觉是吉祥物人物,而非文本框。
4. 能明显感受到 AI 在听/在想/在追问(人物 4 状态)。
5. 每轮只有一个明确学习目标。
6. 练习过程不制造过多纠错压力(无"测试/扣分"等词)。
7. 报告页给出可执行的下一步建议。
8. 场景卡说明训练收益,且支持自定义。
9. 右侧学习路径显示成长进度而非功能列表。
10. 整体符合"低压开口、真实追问、可见成长"定位,适合自用 + 课堂/商业演示,且结构上留有付费升级空间。

## 范围切分

- **P0(本次)**:三页 UI 重构 + 吉祥物人物升级(SVG)+ 舞台式练习(按钮提交 + 实时字幕)+ 诊断单报告 + 去压力化文案 + 成长轨迹(mock 数据展示)+ 后端 `roundGoal`/`positiveFeedback` 字段。
- **P1(另开 spec)**:静音自动提交的 VAD;即时反馈更丰富的后端编排;付费分层 UI。
- **P2(另开 spec)**:历史练习对比;面试题库;PDF 报告导出。
