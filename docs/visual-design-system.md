# 视觉设计系统：Fluent 2 系统伴侣风格

## 1. 视觉定位

AI Speaking Coach 的首版视觉方向采用 Fluent 2 / Windows 11 系统伴侣风格。产品功能仍然是中文用户优先的英语口语场景冲刺工具，但界面不再走漫画化表达，而是更接近 Windows 11 中的控制面板、小组件和效率工具。

设计目标：

- 保留微软生态的现代、轻盈、系统感。
- 适配 Web 产品的清晰信息结构和响应式布局。
- 让英语练习工具看起来像一个可信赖的“系统级练习助手”，而不是娱乐化页面。
- 在语音练习、纠错、评分等高信息密度场景中保持可读性和稳定性。
- 在 Fluent 2 外壳中加入原创虚拟教练互动层，提升陪伴感和每日复练动力。

## 2. 设计原则

- 现代轻盈：用柔和渐变背景、浅色层次和细腻阴影建立空间感。
- 系统亲和：组件形态贴近 Windows 11 / Fluent 2 的圆角、轮廓、蓝色强调和线性图标。
- 信息优先：纠错说明、评分解释、转写内容必须比装饰更重要。
- 克制动效：交互反馈柔和、短促、稳定，不夸张弹跳。
- 中文优先：导航、按钮、说明、评分解释使用中文，英文只用于学习内容和标签补充。
- 原创角色：虚拟教练必须是原创未来职场教练，不复刻任何现有产品 IP、角色、界面和动效。

## 3. 色彩体系

主色与语义色：

- Fluent Blue：`#2563EB`，用于主按钮、当前状态、关键进度。
- Soft Blue：`#60A5FA`，用于渐变、hover 和轻量强调。
- Cyan Glow：`#67E8F9`，用于语音状态、识别中状态和轻微光感。
- Success Green：`#16A34A`，用于完成状态和正向反馈。
- Warning Amber：`#F59E0B`，用于需注意的表达问题。
- Error Red：`#DC2626`，用于严重错误或识别失败。

背景与中性色：

- App Background：`#F7FAFF`。
- Gradient Start：`#EEF6FF`。
- Gradient End：`#F8FBFF`。
- Card White：`rgba(255, 255, 255, 0.72)`。
- Stroke Light：`rgba(255, 255, 255, 0.75)`。
- Stroke Subtle：`rgba(37, 99, 235, 0.16)`。
- Text Primary：`#111827`。
- Text Secondary：`#4B5563`。
- Text Muted：`#6B7280`。

使用原则：

- 页面背景使用浅色蓝白渐变，不使用高饱和大色块。
- 主行动只使用 Fluent Blue，避免多个强色抢焦点。
- 评分图表可使用蓝、绿、黄、红，但必须保持语义一致。
- 透明卡片必须有足够对比度，正文区域不能过度透明。

## 4. 材质与层次

核心材质是亚克力卡片：

- 卡片背景：高透明度白色叠加 `backdrop-filter: blur(18px)`。
- 卡片边框：1px 半透明白色或淡蓝描边。
- 阴影：柔和、大半径、低透明度，例如 `0 20px 60px rgba(15, 23, 42, 0.10)`。
- 圆角：主卡片 20-28px，普通按钮 12-16px，小标签 999px 胶囊。

层次结构：

- 背景层：柔和渐变、极轻纹理或模糊光斑。
- 页面层：内容容器，限制最大宽度并保持充足留白。
- 卡片层：首页任务卡、场景卡、练习房间、报告卡。
- 控件层：按钮、分段控件、进度条、状态标记。
- 反馈层：toast、加载状态、识别中提示。

## 5. 布局语言

首版采用稳定、清晰的工具型布局。

### 首页

- 顶部使用简洁导航和产品标识。
- 主区使用大亚克力卡片展示“今日推荐练习”。
- 右侧或下方展示虚拟教练欢迎卡、最近得分、连续练习、本周打卡和推荐场景。

### 场景库

- 使用三张柔和卡片展示“面试 Interview”“会议 Meeting”“点餐 Restaurant”。
- 卡片内使用线性图标、场景说明、难度标签、预计时长。
- Hover 时卡片轻微上浮，边框蓝色增强。

### 任务准备

- 使用步骤式面板展示任务目标、AI 角色、练习重点、预计轮次。
- 开始按钮固定为主按钮，文案使用“开始 5 分钟练习”。

### 练习房间

- 左侧：虚拟教练视频舞台，使用亚克力卡片承载原创未来职场教练。
- 主区：对话转写和 AI 当前问题。
- 右侧：任务进度、轮次、轻提示和评分预览。
- 底部或固定区：麦克风按钮、文字模拟输入、结束练习。
- 语音状态使用柔和蓝色发光和进度动画，不使用夸张动效。

### 课后报告

- 顶部展示总分和一句中文总结。
- 顶部或侧栏展示打卡完成卡，显示今日已完成、连续练习天数和本周点亮进度。
- 五维评分使用卡片、条形进度或雷达图。
- 逐句纠错使用表格式卡片：原句、推荐表达、中文原因。
- 改进建议以 3 张优先级卡片展示。
- 虚拟教练点评卡使用高不透明度亚克力面板，保证中文点评可读。

## 6. 组件规范

### 按钮

- 主按钮：Fluent Blue 背景、白字、12-16px 圆角。
- 次按钮：半透明白底、淡蓝边框、深色文字。
- 危险操作：只用于结束或清空，使用红色但避免大面积使用。

### 卡片

- 使用亚克力背景和细描边。
- 卡片内间距至少 20px。
- 信息密集卡片使用更高不透明度，保证中文阅读清楚。

### 分段控件

- 用于场景切换、报告维度切换。
- 当前项使用蓝色填充或浅蓝背景。

### 状态标记

- 胶囊形状，用于“识别中”“第 3/6 轮”“轻提示已开启”。
- 不使用过多颜色，优先蓝/绿/黄三类。

### 打卡小组件

- 使用 Fluent 2 小组件形态：亚克力卡片、胶囊状态、7 天横向点亮进度。
- 今日完成状态使用蓝色或绿色强调。
- 连续天数以大数字展示，旁边配中文解释，例如“连续练习 4 天”。
- 不使用金币、商店、排行榜、补签卡或复杂徽章体系。

### 虚拟教练视频舞台

- 舞台是大圆角亚克力卡片，背景使用轻微蓝白渐变和柔和光斑。
- 角色为原创未来职场教练，建议蓝白科技感服装、利落发型、专业但有气场。
- 舞台下方或右上角显示状态胶囊，例如“倾听中”“思考中”“追问准备”。
- 角色区域不能遮挡对话转写、麦克风按钮或任务进度。

### 进度与评分

- 进度条使用柔和宽度过渡。
- 五维评分保留中文主标签和英文副标签：
  - 发音 Pronunciation
  - 流利度 Fluency
  - 语法 Grammar
  - 表达 Expression
  - 任务完成 Task Completion

### 虚拟教练状态

- 待机：轻微呼吸和眨眼。
- 倾听：录音中角色保持看向用户，显示蓝色识别光环。
- 思考：AI 生成追问时显示小型 shimmer 或点状加载。
- 追问：AI 推进下一轮问题时切换为聚焦表情。
- 评价：课后报告中给出建设性点评。
- 完成庆祝：打卡成功时使用轻量庆祝动效，不使用夸张弹跳。

## 7. 字体与图标

字体：

- 中文优先使用系统无衬线字体：`Microsoft YaHei UI`、`PingFang SC`、`Noto Sans SC`、`system-ui`。
- 英文和数字使用系统字体，确保和 Windows / Web 端都协调。
- 标题使用 600-700 字重，正文使用 400-500 字重。

图标：

- 使用线性图标风格，笔画统一，避免填充式卡通图标。
- 场景图标可使用 briefcase、users、utensils、mic、sparkles、chart 等语义图标。
- 图标颜色以 Fluent Blue 或中性灰为主。

## 8. 动效与交互

动效原则：

- 时长：150-300ms。
- 缓动：柔和 ease-out 或 Fluent 风格曲线。
- 位移：轻微上浮 2-4px，避免大幅弹跳。
- 阴影：hover 时阴影略增强。
- 录音状态：使用柔和脉冲或环形进度。

关键状态：

- Hover：卡片上浮、边框蓝色增强。
- Pressed：按钮颜色加深，位移回落。
- Loading：AI 思考中显示小型 shimmer 或点状加载。
- Recording：麦克风按钮出现蓝色光环和计时。
- Completed：报告卡淡入，总分数字平滑递增。
- Check-in completed：本周点亮进度柔和填充，连续天数数字短暂高亮。

## 9. 中文用户适配

- 页面导航、按钮、提示、评分解释、错误原因使用中文。
- 英语学习内容保留英文原句，并配中文解释。
- 场景命名使用“中文 + 英文标签”，例如“面试 Interview”“会议 Meeting”“点餐 Restaurant”。
- AI 角色使用中文说明，例如“AI 面试官将追问你的项目经历”。
- 虚拟教练使用中文短句进行督促和反馈，例如“别绕，先把项目结果说清楚。”
- Demo 路线用中文叙述，保证评委无需理解全部英文也能看懂产品价值。

## 10. 可读性与遮挡检查

每个页面交付前必须检查：

- 桌面宽屏、普通笔记本、移动端三类视口下无文字重叠。
- 中文按钮、标签、任务卡标题完整显示。
- 亚克力透明度不能导致正文对比度不足。
- 长中文纠错原因必须自然换行，容器高度随内容增长或提供滚动。
- 录音中、AI 思考中、hover、报告生成等状态不能造成布局跳动。
- 虚拟教练的待机、倾听、思考、追问、评价、完成庆祝状态不能遮挡转写、按钮、评分和中文解释。
- 打卡小组件在桌面和移动端都要完整显示 7 天点亮进度。
- 截图检查覆盖首页、场景库、任务准备、练习房间、课后报告。

## 11. 不采用的方向

首版不采用漫画书风格、游戏化重装饰、夸张爆炸图形、粗黑描边和大量倾斜分格。原因是该方向记忆点强，但在中文纠错、评分解释和 Web 产品可复现方面不够实用。

同时不采用完整游戏化体系，包括金币、商店、排行榜、补签卡、复杂等级和非原创角色模仿。连续打卡只作为轻量复练反馈存在。
