# UI 信息架构

## 1. 导航结构

首版采用单页应用结构，核心页面控制在 4 个，保证 demo 清晰、实现可控。

```text
首页
  -> 场景库
      -> 任务准备
          -> 练习房间
              -> 课后报告
```

辅助入口：

- Home 可直接进入推荐任务。
- Session Report 可返回 Practice Room 再练一次，也可回到 Scenario Library 切换场景。

## 2. 页面职责

### 首页

目标：让用户快速开始一次练习。

主要模块：

- 顶部产品标识和当前练习目标。
- 快速开始卡片。
- 虚拟教练欢迎卡：展示今日任务、连续练习天数和一句中文训练提醒。
- 本周打卡进度：7 天点亮小组件，突出今日是否完成。
- 推荐场景卡片。
- 最近一次练习结果。
- 今日建议，例如“先完成一次面试自我介绍”。
- AI 练习助手的中文引导卡片。

### 场景库

目标：让用户选择练习场景和任务类型。

主要模块：

- 场景分类：面试 Interview、会议 Meeting、点餐 Restaurant。
- 场景说明：AI 角色、适合目标、难度。
- 任务卡列表：每张卡包含目标、预计时间、练习重点。

### 任务准备

目标：在开始语音前明确任务边界。

主要模块：

- 任务目标。
- AI 角色设定。
- 虚拟教练任务说明：用中文说明本轮目标、节奏和注意点。
- 本轮练习重点。
- 预计轮次和时长。
- 开始练习按钮。

### 练习房间

目标：完成 4-6 轮角色对话练习。

主要模块：

- 左侧虚拟教练视频舞台：展示原创未来职场教练，支持待机、倾听、思考、追问、评价状态。
- 主对话区：展示当前问题或追问、用户实时转写、AI 回复。
- 右侧控制面板：展示任务目标、轮次进度、轻提示、练习重点和评分预览。
- 底部控制区：语音输入按钮、识别状态、文字输入 fallback、结束练习按钮。

### 课后报告

目标：让用户理解本次表现，并知道下一步怎么改。

主要模块：

- 打卡完成卡：显示今日已完成、连续练习 N 天、本周 7 天点亮进度。
- 总分。
- 五维评分。
- 表现摘要。
- 逐句纠错。
- 推荐表达。
- 3 条优先改进建议。
- 虚拟教练点评卡：用中文总结本轮表现，并给出下一次训练建议。
- 再练一次 / 换场景按钮。

## 3. 关键交互状态

- 未授权麦克风：提示授权，并提供文字模拟输入作为 fallback。
- 虚拟教练待机：首页或任务准备页保持轻微呼吸、眨眼状态。
- 录音中：主按钮显示录音状态和时长。
- 虚拟教练倾听：录音中进入倾听状态，避免遮挡转写。
- AI 思考中：显示正在分析/追问状态，避免用户误以为卡住。
- 虚拟教练思考：AI 生成追问时显示短暂思考状态。
- 用户卡住：轻提示区给出一句候选表达。
- 虚拟教练追问：AI 继续推进任务时进入追问状态。
- 课后完成：虚拟教练进入评价或完成庆祝状态，同时点亮今日打卡。
- 练习结束：锁定本轮记录，进入报告页。

## 4. Demo 优先路径

首版实现优先保证下面路径顺畅：

```text
首页 -> 场景库 -> 面试任务 -> 练习房间 -> 课后报告
```

会议和点餐在首版中也需要可选择，但 demo 主路线优先使用面试。
